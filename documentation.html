FriendMap Documentation

Stay connected with friends worldwide through visual friendship mapping and smart travel planning.

Overview

FriendMap is a lean web application designed to help you maintain friendships across the globe while making travel planning more social and cost-effective. It combines friendship management, contact tracking, and travel planning into one intuitive tool.

Core Philosophy

In our increasingly connected yet busy world, maintaining meaningful friendships requires intentional effort. FriendMap bridges the gap between:

  • Friendship maintenance: Visual reminders to stay in touch
  • Travel planning: Leveraging your social network for accommodation and local insights
  • Global awareness: Understanding where your friends are and when you last connected

Getting Started

First Time Setup

  1. Open index.html in your web browser
  2. The app will automatically load with demo data to show you how it works
  3. Navigate to the "Friends" tab to start adding your own friends
  4. Your data is automatically saved in your browser's local storage

Data Model

FriendMap uses a simple JSON data structure to store friend information:

{ "id": 1, "name": "Friend Name", "location": "City, Country", "coordinates": { "lat": 52.5200, "lng": 13.4050 }, "lastContact": "2024-06-15T00:00:00.000Z", "canStay": true, "notes": "Personal notes about accommodation or relationship" }

Field Descriptions

  • id: Unique identifier (auto-generated)
  • name: Friend's name (required)
  • location: City and country in "City, Country" format (required)
  • coordinates: Latitude and longitude (auto-generated via geocoding)
  • lastContact: Date of last contact (required)
  • canStay: Boolean indicating accommodation availability
  • notes: Free-form text for additional details

Data Storage

All data is stored locally in your browser using localStorage. This means:

  • Your data stays private and local to your device
  • No server required - the app works offline
  • Data persists between browser sessions
  • Clearing browser data will remove your friends list

Usage Guide

Adding Friends

  1. Go to the "Friends" tab
  2. Fill in the friend form with required information
  3. Check "Can stay with them" if accommodation is available
  4. Add notes about specific details (spare room, cat-sitting, etc.)
  5. Click "Add Friend" to save

Understanding Map Markers

  • Green markers: Recent contact (within 30 days)
  • Orange markers: Overdue contact (30-90 days)
  • Red markers: Needs attention (90+ days)
  • üè† House icon: Accommodation available

Contact Tracking

The "Check-ins" tab shows friends you haven't contacted recently:

  • Friends contacted 30+ days ago appear as reminders
  • Friends contacted 90+ days ago are marked as urgent
  • Use "Mark as contacted today" to update the last contact date

Travel Planning

The "Travel Plans" tab helps you identify travel opportunities:

  • Countries with multiple friends are highlighted
  • Accommodation options are clearly indicated
  • Use filters to focus on specific types of opportunities

Tips & Best Practices

Effective Friend Management

  • Be specific with locations: Use "City, Country" format for best geocoding results
  • Update contact dates regularly: Mark friends as contacted when you reach out
  • Use detailed notes: Include specific accommodation details and relationship context
  • Regular reviews: Check the reminders tab weekly to stay on top of relationships

Travel Planning Strategy

  • Plan around clusters: Visit multiple friends in the same region
  • Lead with accommodation: Start planning with friends who can host you
  • Consider timing: Reach out to overdue contacts when planning travel
  • Document connections: Use notes to track introductions between friends

Neurodiversity-Friendly Features

  • Visual cues: Color-coded system reduces cognitive load
  • Structured data: Clear categorization helps with organization
  • Reminders: Automatic tracking reduces mental overhead
  • Flexible notes: Accommodate different communication preferences

Technical Details

Technology Stack

  • Frontend: HTML5, CSS3, Vanilla JavaScript
  • Mapping: Leaflet.js with OpenStreetMap tiles
  • Geocoding: Nominatim API (OpenStreetMap)
  • Storage: Browser localStorage
  • Styling: Modern CSS with custom properties

Browser Compatibility

FriendMap works in all modern browsers that support:

  • ES6 JavaScript features
  • CSS Grid and Flexbox
  • localStorage API
  • Fetch API

Privacy & Security

  • Local-only storage: No data sent to external servers
  • Geocoding: Only city names sent to OpenStreetMap for coordinates
  • No tracking: No analytics or user tracking implemented
  • No authentication: No accounts or personal data collection
